## Revival on a new Mac

1. Clone the repo: `git clone <remote> ~/cicero`
2. Add `ANTHROPIC_API_KEY=sk-ant-...` to `~/cicero/.env`
3. Run `./deploy/mac/setup.sh` — idempotent; handles Homebrew, Node, OpenClaw, uv, MCP registration, launchd units, and gateway startup
4. Re-ingest memory: `cd ~/cicero && uv run python scripts/ingest_memory.py`
5. Smoke test: `cicero ask "What's your name and what tools do you have?"`
6. Re-share Apple lists (one-time, from Carlos's phone):
   - Reminders: share Honeydew and Groceries with `cicero.ortega@icloud.com` and Sarah; share Garden with `cicero.ortega@icloud.com` only
   - Notes: share the "Cicero" folder with `cicero.ortega@icloud.com` and Sarah
7. Set macOS permissions (see `docs/operations.md` → "One-time setup"):
   - Full Disk Access for node and `imsg`
   - Reminders permission for node and Terminal
   - Automation permission for Terminal and node (Notes.app, Messages.app)

Full runbook: `docs/operations.md`.

## Revival on Linux (Saturn)

The core of Cicero — personality, memory, model escalation — ports to Linux without code changes.
The Apple integrations need to be replaced. Do this work when Saturn revival is imminent; do not
design it in advance.

### What works as-is

- `lib/memory_mcp.py` and `lib/memory_query.py` — no changes
- `lib/brain_mcp.py` — one-line fix: change `LOG_PATH` from `~/Library/Logs/` to `~/.local/share/cicero/logs/`
- `workspace/skills/cicero-memory/` — no changes
- `workspace/skills/cicero-bigbrain/` — no changes
- `scripts/ingest_memory.py` — no changes
- `pyproject.toml` — all deps are portable
- All `workspace/` identity and personality files — no changes

### What needs to be built

1. **Primary channel** — iMessage has no Linux equivalent. Choose one and implement:
   - Matrix (self-hostable, bridges to many platforms)
   - Signal-CLI (requires phone number; open source)
   - Telegram bot (easiest OAuth; not self-hosted)
   - Simple webhook (least friction; no mobile UX)

2. **cicero-reminders replacement** — EventKit is macOS-only. Options:
   - Todoist API (hosted)
   - Nextcloud Tasks / CalDAV (self-hosted)
   - Design a shared list format that matches the Reminders workflow

3. **cicero-notes replacement** — Apple Notes / osascript is macOS-only. Options:
   - Nextcloud Notes
   - Joplin server
   - Self-hosted wiki (Outline, Obsidian Sync alternative)

4. **Service management** — replace launchd with systemd user services:
   - `~/.config/systemd/user/cicero-gateway.service`
   - `~/.config/systemd/user/cicero-chroma.service`
   - Token rotation via systemd timer instead of `ai.cicero.token-rotate.plist`

5. **Deploy script** — write `deploy/linux/setup.sh` mirroring `deploy/mac/setup.sh`:
   - apt/dnf/pacman instead of Homebrew
   - systemd instead of launchd
   - No `imsg`, no Apple-specific permissions steps

### Sequence for Saturn revival

1. Install OpenClaw, uv, Node on Saturn
2. Clone repo, add `.env`
3. Start Chroma manually or via systemd
4. Register the portable MCPs (memory, brain)
5. Pick and implement a primary channel
6. Once channel works end-to-end, add the notes/reminders replacements
7. Write `deploy/linux/setup.sh` to codify the working configuration
8. See `docs/roadmap.md` — "Saturn migration (deferred)" for the full backlog
